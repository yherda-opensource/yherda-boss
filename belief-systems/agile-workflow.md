# Belief System — Agile Workflow

**Version:** 0.1
**Status:** Emerging — defaults; intended to be adapted per team
**Tags:** #belief-system #agile #workflow #opinionated

---

## Prime Directive

Opinionated defaults for how agile beliefs translate into concrete workflow. These are proposals, not impositions — every team adapts them. The bootstrap discovery arc surfaces which defaults hold and which need replacement.

---

## Core Beliefs

### 1. Ready means the belief is complete enough to act
Work enters Ready when the design is sound, the approach is agreed, and the belief is strong enough to survive implementation without losing fidelity. Moving to Ready is not scheduling — it is a fidelity gate.

### 2. Done means the belief transferred
Work is Done when the output can be decompressed by the receiver with the same fidelity the sender intended. A feature that ships but isn't understood is not Done. A decision that was made but not communicated is not Done.

### 3. Workflow states are belief states
| State | What it means |
|---|---|
| Backlog | Identified — belief not yet formed |
| To Do | Belief formed — not yet ready to act |
| Ready | Belief complete — ready to act |
| In Progress | Acting on the belief |
| Review | Validating fidelity of output |
| Done | Belief transferred successfully |

### 4. Four ceremonies cover the synchronization surface
Standup, demo, planning, and retro cover all four belief synchronization needs: daily progress, delivery validation, next-work alignment, and operational drift. Additional ceremonies are a signal that one of these four isn't working.

### 5. Ceremony cadence defaults
These are starting points. The retro is where the team decides if these hold.

| Ceremony | Default cadence |
|---|---|
| Standup | Daily |
| Planning | Start of each sprint (1–2 weeks) |
| Demo | End of each sprint |
| Retro | End of each sprint, after demo |

### 6. A sprint is a belief commitment, not a time box
The sprint boundary is when the team validates whether what they believed at planning is still true. Short sprints surface drift faster. Long sprints let it compound.

---

## Retro Protocol

1. What felt wrong? — surfaces drift
2. What belief didn't hold? — identifies the gap
3. What do we update? — closes the gap
4. What worked? — reinforces load-bearing beliefs

Output: updated belief system files. Not action items — belief corrections.
