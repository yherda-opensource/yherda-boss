# Bootstrap Story

**Version:** 0.2 (draft)
**Status:** Emerging

A story about constructing a belief-driven workspace. When the final arc completes, the bootstrap dissolves and the workspace takes over.

## Belief Systems

- [[belief-systems/bootstrap-protocol]] — governs how the bootstrap proceeds; every beat produces an artifact; bootstrap dissolves when complete
- [[belief-systems/yherda-setup]] — three-workspace model; baseline, initial, ideal; CLAUDE.md shim
- [[belief-systems/agile-setup]] — retro-first approach to agile instantiation; defaults are proposals

## How to Run This Bootstrap

Paste the contents of this story folder as context. Begin with Arc 1. Each arc completes before the next begins. The active identity is named at the start of each beat.

When Arc 6 completes (or when the user is not yet migrating to Yherda, after Arc 5): clear bootstrap context, load the workspace as prime context.
